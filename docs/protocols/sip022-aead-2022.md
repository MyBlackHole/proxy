# SIP022 AEAD-2022 Ciphers Reference

> Source: https://shadowsocks.org/doc/sip022.html

## Overview

Shadowsocks 2022 Edition defines new AEAD ciphers that replace the old `EVP_BytesToKey` key derivation with BLAKE3-based key derivation. Requires pre-shared keys generated via `openssl rand -base64 <key_size>`.

## Cipher Methods

| Method | Key Bytes | Salt Bytes |
|--------|-----------|------------|
| `2022-blake3-aes-128-gcm` | 16 | 16 |
| `2022-blake3-aes-256-gcm` | 32 | 32 |
| `2022-blake3-chacha20-poly1305` | 32 | optional |
| `2022-blake3-chacha12-poly1305` | 32 | optional |
| `2022-blake3-chacha8-poly1305` | 32 | optional |

## Key Derivation

PSK is base64-encoded binary key (NOT password-derived).

```
session_subkey := blake3::derive_key(
    context: "shadowsocks 2022 session subkey",
    key_material: key + salt
)
```

## TCP Protocol

- 1:1 connection to proxy mapping
- Request stream: salt(16/32B) + 2 standalone header chunks + length chunks + payload chunks
- Response stream: salt(16/32B) + 1 fixed header chunk (also acts as first length chunk)
- Nonce: 12-byte little-endian counter, incremented after each operation

### Request Header (fixed)
```
type(1B) = 0 | timestamp(u64be) | length(u16be)
```

### Request Header (variable)
```
ATYP(1B) | address(var) | port(u16be) | padding_length(u16be) | padding(var) | initial_payload(var)
```

### Response Header (fixed)
```
type(1B) = 1 | timestamp(u64be) | request_salt(16/32B) | length(u16be)
```

### Key Parameters
- Max payload chunk size: 0xFFFF (65535) bytes (vs 0x3FFF in legacy AEAD)
- Timestamp tolerance: 30 seconds for replay protection
- Salt storage: 60 seconds, no Bloom filters (no false positives allowed)

## UDP Protocol

- Session-based with unique session ID used as salt for subkey derivation
- Packet ID counter per session
- Separate header (16B): 8B session ID + 8B packet ID (big-endian)
- AES block cipher for separate header encryption with PSK
- AEAD cipher (AES-GCM) for body with session subkey

### UDP Separate Header
```
session_id(8B) | packet_id(u64be)
```

### Replay Protection
- Sliding window filter per session
- Server routes by client session ID (not source address)
- Session timeout: at least 60 seconds
- Supports client network changes via last-seen-address tracking

## ChaCha Variants (UDP)

For `2022-blake3-chacha20-poly1305`:
- Uses XChaCha20-Poly1305 directly with PSK + random nonce
- 24B nonce prepended to each packet
- Session ID and packet ID merged into main header (not separate)
